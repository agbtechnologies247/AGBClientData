This lets your Rust parser discover **data that is hidden from the visible page but still shipped to or exposed through the frontend**—without attempting to break authentication or access genuinely private data.

For example, a page may visually show:

```text
Product: iPhone
Price: ₹80,000
```

But the frontend may also expose:

```json
{
  "product_id": "123",
  "internal_category_id": "456",
  "stock_count": 17,
  "supplier_code": "ABC",
  "feature_flags": {
    "new_checkout": true
  }
}
```

Your parser should classify those separately:

```json
{
  "visibility": "embedded",
  "access": "public_frontend",
  "source": "window.__INITIAL_STATE__",
  "classification": "frontend_exposed"
}
```

That is the correct boundary for a **public URL/frontend reverse parser**.
